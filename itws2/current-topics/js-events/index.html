1 Objective

The objective of this section is to present two types of "events":

  • User interactions
  • Network interactions

In each case, we specify what is to happen once such an event occurs.

2 Browser Events and Handlers

2.1 Introduction to events

A (DOM) event signifies some kind of interaction of the browser with its surrounding. The interaction may either be triggered by a user event (click of a mouse, pressing of a button, etc.), or may be the result of loading a page etc.

A full classification of events is described in the Mozilla Development Network tutorial on Events:

https://developer.mozilla.org/en-US/docs/Web/Events

2.2 DOM elements, events and handlers

Every DOM element is associated with a set of events, e.g., a button click. A handler, or listener, is a function of one argument (the event) that is invoked after the occurrence of the event. An occurrence of the event causes a thunk invoking the handler with the event to be placed on the Javascript message queue for execution.

An event and listener pair is attached or added against a particular DOM element.

5.3 Javascript

5.3.1 Promises and asynchronous network calls

The getIP function defined below makes a get call to https://api.ipify.org which is a website that returns the IP of the browser's machine. The call $get("https://api.ipify.org") returns immediately. It doesn't wait for website to respond.

The returned object (denoted byhow to get the session on flask server in the front end variable p in the code below) is called a promise.

The done and fail methods on a promise attach functions that specify what to do if the promise's call succeeds (is done) or fails. 

// NOTE: will need to run chrome with cross origin requests enabled
// which is considered unsafe.

// linux: chromium-browser --disable-web-security --user-data-dir
// mac os: open /Applications/Google\ Chrome.app --args --disable-web-security

function getIP() {
  var obj = {ip: "undefined"};

  var p = $.get("https://api.ipify.org/");

  p.done(function(data, responseText, xhr) { 
    console.log("GET Succeeded");
    console.dir({data: data, 
                 responseText: responseText, 
                 xhr: xhr});
    obj.ip = data; 
  });
  
  p.fail(
    function(xhr, responseText, error) { 
      obj.ip = "GET failed"; 
      console.log("GET Failed");
      console.dir({error: error, 
                   responseText: responseText, 
                   xhr: xhr});
    });
  
  var ans = {obj: obj, p: p};
  return ans;
};

5.3.2 Promises vs. synchronous function calls

This function logIPAddress issues a call getIP() which is asynchronous and returns immediately. However, the loopFor calls takes a full 20 seconds before it returns. This call is synchronous.

During this time, the console will block (try it).

Any expression typed will be evaluated after the console is available again. 

function logIPAddress() {
  function loopFor(millisec) {
    var now = new Date().getTime();
    while(new Date().getTime() < now + millisec) { 
      /* no op */
    }; 
  };

  var ans = getIP();

  console.log("long job started...");
  
  loopFor(1000 * 20); // 20 seconds
  // all console 
  // interactions queued
  // up during this time
  console.log("long job ended.");
  return ans;
}
