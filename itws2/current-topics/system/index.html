1 Introduction

We start by considering closed and open systems.

1.1 Closed Systems

  • Until now we worked with closed systems.
  • Our evaluators took an environment and an expression (and perhaps a store) and evaluated the expression in that environment.
  • All that was needed for evaluation was supplied right at the beginning.
  • Such systems are called algorithmic systems.

1.2 Open Systems

  • However, many real world systems continue to receive inputs as they are running.
  • Such systems may also emit an output as part of their execution.
  • Such system are called open or reactive systems.
  • Examples: Human Body, Ecosystem, Stock Market, Web Servers, etc.

2 What is a system?

2.1 Informal Notion

Informally, a system is something that takes inputs, maintains state, and emits outputs. An input event may cause the system's state to change. This change is described by an update function.

2.2 Standard definition

  • A tuple

\[S = \pair{X, X^0, U, f:[X,U]\rightarrow X, Y,h:X\rightarrow Y}\]

where

  • \(X\) is the state space.
  • \(X^0\subseteq X\) is the initial space.
  • \(U\) is the input alphabet, i.e., the set of possible inputs.
  • \(f:[X,U]\rightarrow X\) is the update function.
  • \(Y\) is the output space.
  • \(h:X\rightarrow Y\) is the view function.

2.3 Example: a simple counter

Suppose we want to build a machine that has two input buttons: inc (for increment) and reset for resetting to 0.

The output of the machine is either red or green.

The machine's internal state keeps track of a counter p that is a natural number. Initially, p is zero. Each time the button inc is pressed, the count p increases by one. If p reaches 10, then the output is red, otherwise it is green. Once p reaches 10, any subsequent inc inputs have no effect. Pressing a reset button immediately changes p to zero.

This machine may be formalized as follows:

  • \(U = \set{inc, reset}\)
  • \(X = \set{0, \ldots, 10}\)
  • \(X^0 = \set{0}\),
  • \(f(x,{\texttt{inc}}) = x, \mbox{ if } x=10, \mbox{ and } x+1, \mbox{ otherwise}\) and
  • \(f(x,{\texttt{reset}}) = 0\).
  • \(h(x) = \texttt{red}, \mbox{ if } x = 10, \mbox{ and } \texttt{green}, \mbox{ otherwise}\).

2.4 Actors

An actor is a device with an internal state. It receives input messages. Upon receiving a message, an actor updates its state and may also send a message to other actor(s).

It's clear that an actor is a system.

2.5 Interconnected Systems

A system is interconnected to another system if it shares its input or output with the input or output with another system.

The interconnections introduce constraints between the dynamics of two systems.

  • Often systems are connected to each other to form bigger systems.
  • Example: CPU system connected to the memory system.
  • Example 2: Computers connected in a network.
  • Example 3: Web browser and its interconnected systems.
