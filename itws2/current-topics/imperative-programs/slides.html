3 Components of the imperative model

3.1 Locations and Memory

  • Memory indexed with locations, which are natural numbers.
  • Memory maps Locations to Values 
Mem = Loc -> Val

3.2 Environments redefined

Env = Id -> Loc

3.3 No pointers

  • Locations are not values.
  • Contrast this with C:
    • Procedures are not values.
    • Locations are values (pointers)

3.4 Memory, like environment is implicit

  • Memory is not explicit
  • Can not be returned as a value
  • Similar to Environment in this respect.

3.5 Identifier vs. Variables

A variable is an identifier whose denotation is a location rather than a value.

3.6 Imperative program syntax

We extend the syntax of pure functional programs with the following productions 

<exp> ::= 
          <assign-exp> |
          <seq-exp>

<assign-exp> ::= (set! <id> <exp>)
<seq-exp>    ::= (begin <exp> <exp> ...)

3.7 Assignment occurrence

Assignment occurrence
An identifier occurring in a set! expression after the set! keyword.

4 Evaluating imperative programs

4.1 Basic structure of evaluation

  • Evaluating an imperative programs also returns a value.
  • But may also side-effect, or modify the state.
  • How do we characterise the signature of evaluation in the imperative setting?

4.2 The signature of the evaluator

  • Takes an expression, an environment and a memory
  • Returns a value, and also a memory
  • The memory could potentially be a new mapping from locations to values. 
eval: Exp, Env, Mem -> [Val, Mem]

4.3 Variable lookup

In the imperative model, evaluating a variable reference occurrence involves the following steps:

  • Look up the variable in the environment.
  • The lookup results in a location
  • Take the location and index it into the store.

4.4 Variables denote locations

Declarative model
Evaluation of reference occurrence of an identifier returns the identifier's denotation (a value).
Imperative model
Evaluation of reference occurrence of an identifier does not return the identifier's denotation (a location).

Instead it returns the contents of the memory indexed by the location.

4.5 Assignments modify memory

(set! <id> <exp>)
  • Updates the memory's location denoted by <id> to the value obtained by evaluating <exp>.
  • Returns the value void.
