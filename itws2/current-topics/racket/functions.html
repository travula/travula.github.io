1 Functions

1.1 Anatomy of a function

  • keyword
  • formals
  • body

A function expression is a list with 3 elements. The first is a keyword lambda. The second is a list of formals. The third, called the body is any racket expression. The BNF (Backus-Naur Form) syntax of a function expression is given below. <formal> ... means zero or more formals. 

<function-exp> ::= (lambda (<formal> ...) <body>)
<formal> ::= <id>
<body>  ::= <exp>

The result of evaluating a function is a procedure that takes as many arguments as specified by length of formals. You can't look inside a procedure. You can, however, apply it to arguments.

1.2 Function application

In the syntax of an application function, which is given below, the first expression must evaluate to a function. 

<application-exp> ::= (<exp> <exp> ...)

2 Example functions

(require racket/math) ; pi's definition is in the math package

(define area
  (lambda (radius)
    (* pi radius radius)))

(area 3.0) ; => 28.274333882308138

3 Binding during application

When an application is evaluated, the operator expression, which is the first expression is evaluated. This evaluation results in a procedure. The formal arguments of the procedure bound to the arguments obtained by evaluating the operand expressions in the application.

3.1 The define shortcut

Sometimes the following syntax is used for functions: 

(define (twice x)
  (* 2 x))

This is a shortcut for 

(define twice
  (lambda (x)
    (* 2 x)))

In this course we will always use the second form because that form emphasises the distinction between creating a function and binding an identifier to it.
