1 Objective

The objective of this activity is to encourage students for contributing to the content of the course. Students often refer to online content such as tutorials, videos, write-ups, blogs etc. for self learning. Also, many students try out examples that they find interesting or help them in understanding certain concepts better. Such examples are also welcome for contribution. The result of this activity is to gather a pool of information that is relevant to the course and for fellow students to make use of.

Added bonus: Students also learn how to create merge requests and other subtle nuances of open source contribution.

Other takeaways
  • Learn to collaborate and work in teams
  • Resolving merge conflicts
  • Adhering to the given instructions :P

4 Submission Instructions

IMPORTANT NOTE
Each group needs to co-ordinate and decide only one person to be the one to create a merge request.
  1. Each group is given a topic. You need to collect any resources/tutorials/any helpful material that you find relevant to the course and to your topic.
  2. Add the content that you found into the <topic-name>.org file.
  3. All students will have to fork this repo and make a merge request to make contributions to this repository.
  4. To fork, follow the instructions mentioned here.
  5. After forking, clone your repo. 
    git clone https://gitlab.com/<username>/course-forum.git
$ cd course-forum
  6. Make your changes in this repo i.e. <username>/course-forum.You'll now have to synchronize with the main repository, so add an additional remote. We'll call the main repository upstream from now on. 
      $ git remote add upstream https://gitlab.com/itws2-2017-spring-iiith/course-forum.git
  7. After committing, do a pull from upstream to synchronize your repo with any changes that might have been made upstream. 
    $ git pull upstream master
  8. Any merge-conflicts would show up at this stage if a file in your repo conflicts with the upstream master branch. Fix these by modifying the conflicting files and commit any changes. 
    $ # Fix merge conflicts
$ git commit -m "Fixed merge conflict: <more description>"
  9. Now push the changes to the master branch in your fork. 
    $ git push origin master
  10. Make sure your commit messages and merge request messages are descriptive enough.
  11. Now head over to the gitlab UI, and from your fork of the repo create a merge request to itws2-2017-spring-iiith/course-forum. Your commits will get reviewed by the TAs over there and will get accepted if satisfactory. If not, we'll respond with what modifications to make in comments on your merge request.

If you are googling for help/details on any of the terms mentioned above, please keep in mind that a merge request in gitlab is equivalent to a pull request (PR) in github terms.

5 Instructions

  1. This is a contributory activity and not a competitive one. Please coordinate with your fellow group members (of the same topic) and make sure there are no duplicates/redundancies
