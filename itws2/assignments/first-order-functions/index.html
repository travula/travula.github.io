1 Problems

Design Programs for each of the following problems.

1.1 vector-sum

1.1.1 Problem

Write a function vector-sum which takes a vector v of numbers, and returns the sum of all elements of v.

1.1.2 Test cases

(check-equal? (vector-sum #()) 0 "test 1")
(check-equal? (vector-sum #(1 2 3)) 6 "test 2")
(check-equal? (vector-sum #(-1 2 3)) 4 "test 3")

1.2 list-sum

1.2.1 Problem

Write a function called list-sum which takes a list ln of numbers and returns the sum of all the numbers in ln.

1.2.2 Test cases

(check-equal? (list-sum '()) 0 "test 1")
(check-equal? (list-sum '(1 2)) 3 "test 2")
(check-equal? (list-sum '(1 2 8 4)) 15 "test 3")

1.3 list-length

1.3.1 Problem

Write a function called list-length which takes a list and returns the number of elements in it.

1.3.2 Test cases

(check-equal? (list-length '()) 0 "test 1")
(check-equal? (list-length '(1 2 a)) 3 "test 2")
(check-equal? (list-length '(1 2 (1 2 3))) 3 "test 3")
(check-equal? (list-length '((1 2) (3 4) (5 6))) 3 "test 4")

1.4 member?

1.4.1 Problem

Write a function named member? which takes a list and a value and returns true if the value is equal to any element of the list, false otherwise.

1.4.2 Test cases

(check-equal? (member? '() 'a) #f "test 1")
(check-equal? (member? '(a b c) 'a) #t "test 2")
(check-equal? (member? '("a" "b" "c") 'a) #f "test 3")
(check-equal? (member? '(1 2 3 (5 6)) '(5 6)) #t "test 4")

1.5 map-sqr

1.5.1 Problem

Write a function named map-sqr which takes a list of numbers and returns a list whose elements are squares of the corresponding elements of argument list.

1.5.2 Test cases

(check-equal? (map-sqr '(1 2 3 4)) '(1 4 9 16) "test-1")
(check-equal? (map-sqr '(1 2/3 3.0 3+4i)) '(1 4/9 9.0 -7+24i) "test 2")
(check-equal? (map-sqr '()) '() "test 3")

2 Instructions

2.1 Providing functions

Once you've written these programs, you need to 'provide' them so that they can be evaluated by us. Add the following at the top of your file. 

(provide
    vector-sum
	list-sum
	list-length
	member?
	map-sqr
    )

2.2 Naming convention

  • 1. Use the same names of functions as mentioned in the questions.
  • 2. All the questions should be in the same file, and the file is to be called "main.rkt".

2.3 Directory Structure

  • 1. Each student has access to a repository whose name is the roll-number of the student.
  • 2. In your GitLab repository, you must follow the directory structure as below: 
    -------------------------
<roll-no>
|
`----assignment-solutions
     |
     `--- assignment-1
          |--- main.rkt
------------------------

2.4 Strict Adherence

If you do not adhere to this directory structure and naming, you will lose marks. The evaluation of this homework is automated. So, please make sure that you don't miss hyphens and upper case letters in file, directory and function names. No requests for regrading of your homework will be entertained if the directory structure or naming convention is not followed!
